# Contributing to Armert Labs

Thank you for your interest in contributing to **Armert Labs** open-source projects.

## Before you start

1. Check the target repository's own `CONTRIBUTING.md` if present — it takes precedence.
2. Open an issue for significant changes before submitting a large pull request.
3. Keep pull requests focused; one logical change per PR.

## Development standards

- **Tests** — behavior-changing PRs should include or update tests where applicable.
- **Lint & types** — follow existing tooling in each repo (Ruff, mypy, ESLint, etc.).
- **Security** — never commit secrets, API keys, or `.env` files. Report vulnerabilities via [`SECURITY.md`](SECURITY.md).
- **Commits** — clear messages describing *why*, not only *what*.

## Pull request process

1. Fork the repository and create a feature branch from `main`.
2. Make your changes and ensure CI passes locally when possible.
3. Open a PR with a concise summary and test plan.
4. Address review feedback; maintainers will merge when checks pass.

## Code of conduct

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
